Permission Rationale

Why DeadScroll needs sensitive Android access.

DeadScroll intervenes in real time. That means some Android permissions are not optional if the product is going to work honestly. This page explains what each permission is for, what it enables, and what DeadScroll does not use it to do.

Trust Rule

Explain the power. Explain the limits.

Any app asking for Accessibility or Overlay should be able to justify those requests in plain language. DeadScroll should be no exception.

Core Permissions

These permissions exist because the nudge has to appear at the right moment.

Accessibility

Detect target-app entry and loop signals

Accessibility helps DeadScroll detect when selected target apps open and when relevant interface signals, such as scrolling, show that a loop may be forming.

Overlay

Place the intervention over the app

Overlay permission lets DeadScroll display the nudge above the target app in real time. Without it, the intervention cannot appear where the drift is happening.

Usage Access

Support onboarding and usage-aware surfaces

Usage Access helps DeadScroll show where attention is getting lost and supports certain usage-based parts of the experience.

What DeadScroll Does Not Do

The power of the permission does not define the intent of the product.

Accessibility Limits

DeadScroll is not trying to inspect your private content.

  • It is not built to read your private messages for meaning.
  • It is not built to collect passwords or typed secrets.
  • It is not built to transmit the contents of your screen to DeadScroll-controlled servers.
Usage And Storage Limits

Core app state is intended to remain local.

  • DeadScroll does not require an account for core functionality.
  • Settings and intervention history are intended to remain on-device.
  • Any third-party technical processing should be disclosed separately in the privacy policy.
Control

Users can revoke access at any time.

System Control

Accessibility and Overlay can be turned off from Android settings.

If those permissions are revoked, DeadScroll's core intervention features may stop working until they are restored. That is the expected behavior.

Removal

Deleting the app removes local DeadScroll data from the device.

If a third-party technical service has already processed limited event or diagnostics data in your build, retention may be governed by that provider's own policy.
