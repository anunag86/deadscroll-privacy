Why DeadScroll Exists

Why DeadScroll nudges instead of just blocking

A blocker can shut the door for a while. DeadScroll is trying to catch the moment just before compulsion takes over, because that is where a user still has a real chance to choose differently.

Typical Blocker

Brute interruption

  • Usually relies on blunt lockout instead of contextual intervention.
  • Can feel punitive or mechanical.
  • Can be bypassed, resented, or disabled.
  • Usually treats every target app the same way.
DeadScroll

Contextual intervention

  • Introduces a pause while intention is still recoverable.
  • Escalates only when drift keeps forming.
  • Uses different nudge types for different loop states.
  • Aims for less reactance and more long-term buy-in.
Nudge Ladder

The first nudge is a pause. The next ones get stronger.

DeadScroll does not publish exact trigger thresholds, but the system is not random. The user sees lighter friction first, then stronger interventions if the same loop keeps reforming.

01

Opening a target app can trigger a short pause. This is meant to interrupt autopilot without feeling like a lockout.

02

If drift starts to build, DeadScroll can move into stronger friction before the session hardens into a longer loss.

03

Repeated entry, continued session stretch, or loop behavior can unlock more demanding nudges that force active attention.

04

When resistance keeps repeating, the system becomes firmer. The point is to break hardened looping, not to flatter it.

Current Nudge Modes

Different loops need different kinds of friction.

Pause Friction

Hold

The user presses and holds until a timed toll completes. This creates a pause without adding cognitive clutter too early.

Regulation Friction

Breathe

A guided breath loop slows the state down physically. It is used when the best interruption is regulation, not homework.

Cognitive Friction

Cipher

Alphanumeric input forces active attention so the user cannot slide through a nudge mindlessly and keep scrolling.

Cognitive Friction

Stoic Scribe

Short written prompts make the user re-enter language and intention. This is for moments when the loop needs a harder break.

User Control

You choose the baseline. DeadScroll chooses the moment.

Severity is a user-facing contract, not a hidden punishment setting. It changes how persistent and how fast DeadScroll is willing to escalate.

Severity Mode

Gentle

Slower escalation, lighter pressure, and the most forgiving posture for users who want a conscious pause without a hard clamp.

Severity Mode

Mindful

The balanced default. Enough friction to break the dopamine loop without turning the product into a blocker in disguise.

Severity Mode

Ironclad

Fastest escalation and strongest persistence for users who want DeadScroll to step in earlier and hold the line harder.
